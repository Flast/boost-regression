Regression Test Reporting Tools

This repository is the home of tools used to run Boost's automated regression testing and reporting, and tools that may (optionally) be used by Boost developers to perform local regression test reporting.

Instructions for running Boost's automated regression testing and reporting, and reporting
the results back to the Boost community are available on the Boost web site.

To build developers local test reporting tools, cd to the specific tool's build directory, then invoke b2 like this:

b2 --boost-root=path --boost-build=path/build/src install

For example:

cd regression/library_status/build
b2 --boost-root=c:/boost --boost-build=c:/boost/tools/build/src install
Absolute paths are currently required.

There are two developers local test reporting tools, plus the process_jam_log tool that prepares data for the reporting tools.

  • process_jam_log.cpp - Processes the bjam outputs, creating a file named test_log.xml for each test encountered.
  • Compiler Status - Generates HTML status tables from test_log.xml and other files. Simpler to use than Library Status, but less capable.
  • Library Status - Runs test programs for one or all boost libraries on your local installation and generates complete tables to show which combinations of libraries, compilers, compiler settings pass and fail at your local installation. More capable than Compiler Status, but more complex to use.
